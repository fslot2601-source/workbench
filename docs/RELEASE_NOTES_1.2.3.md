# Workbench 1.2.3

这个版本保留 1.2.2 的状态栏紧凑布局，并修正其在 Xcode 16.2 环境下的源码编译兼容性。

## 本次更新

- 状态栏根据实际内容自动收缩，减少无意义留白。
- 内容较多时保持合理最大高度并支持滚动。
- 高度更新明确在主线程执行，兼容项目 CI 使用的 Xcode 16.2。

## 为什么 1.2.2 会出问题

状态栏此前在界面和系统弹窗两处都固定为 560pt 高。信息较少时，内容已经结束，但外框不会缩小，因此会出现明显的底部留白。1.2.2 引入动态高度正是为了解决这个问题。

第一次实现可以在本地较新版 Xcode 中构建，但项目 GitHub Actions 固定使用 Xcode 16.2。这个版本会把 SwiftUI 的高度监听回调按 `@Sendable` 闭包检查；旧实现直接从该回调更新只允许主线程访问的界面状态，于是云端报出 Swift 并发隔离错误。

1.2.3 没有撤销动态高度，而是把状态更新明确交回 `MainActor`。因此状态栏仍会自动收缩，同时源码已经在 Xcode 16.2 下通过单元测试、界面测试和发布构建验证。

需要特别说明：这次问题属于源码构建兼容性，不是 Workbench、ChatGPT 或 Codex 的运行时连接故障。1.2.2 的本地安装包不因此失效，但为了让下载版本与 GitHub 可复现构建保持一致，1.2.3 是当前推荐版本。

## 下载

- `Workbench-1.2.3-16.zip`：直接解压使用。
- `Workbench-1.2.3-16-SHA256SUMS.txt`：下载校验值。

当前社区构建没有 Apple Developer ID 签名，也没有经过 Apple 公证。首次打开时，请在 Finder 中右键 Workbench 并选择“打开”。
