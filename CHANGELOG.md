# Changelog

## 1.3.0 - 2026-07-16

- 增加公开开发路线，说明近期优先级、后续方向和明确不做的事项。
- 接入 Sparkle 2 自动更新框架，在“设置 → 关于”和应用菜单中增加“检查更新”，并允许关闭自动检查。
- 建立基于 GitHub Release、HTTPS appcast 与 EdDSA 更新签名的社区版发布流程；更新私钥只保存在维护者 macOS 钥匙串。
- 发布验证现在检查 Sparkle 框架、公钥、更新地址和安装包内的符号链接一致性。

## 1.2.3 - 2026-07-16

- 修正状态栏动态高度回调在 Xcode 16.2 下的 Swift 并发编译错误，保持紧凑布局不变并恢复源码与云端构建兼容性。
- 原因是本地较新版 Xcode 与项目 CI 固定使用的 Xcode 16.2 对 SwiftUI 回调的并发隔离检查不同；1.2.2 直接在 `@Sendable` 回调中更新主线程状态，因此本地能够构建、云端却未通过编译。1.2.3 将高度更新明确切回 `MainActor`。

## 1.2.2 - 2026-07-16

- 此版本已由 1.2.3 替代：安装包本身不涉及运行时故障，但对应源码未通过项目的 Xcode 16.2 云端构建门禁。
- 状态栏面板改为根据实际显示内容自动收缩，不再固定保留 560pt 高度；内容较多时限制最大高度并继续支持滚动。
- 压缩状态栏分区、卡片和操作按钮的纵向间距，提高信息密度并减少无意义留白。

## 1.2.1 - 2026-07-16

- 兼容 Codex 整合进 ChatGPT 后的新安装位置，自动发现 `ChatGPT.app` 内置的 Codex，同时继续支持旧版 `Codex.app` 与独立 Codex CLI。
- 设置页现在可以直接选择 `ChatGPT.app`、旧版 `Codex.app` 或 `codex` 可执行文件，并自动解析应用内的实际执行路径。
- 手动选择无效位置、普通文件或缺少执行权限时显示具体原因，不再统一误报为“没有找到 Codex CLI”。
- 增加应用包、独立可执行文件、符号链接与错误路径的回归测试，并通过当前 ChatGPT 内置 App Server 的真实连接验证。

## 1.2.0 - 2026-07-14

- 应用更名为 Workbench，更新 App 图标和状态栏单色工具图标，并保留原 bundle id 以兼容已有本地设置。
- 增加原生状态栏面板，集中显示 Codex 限额、Token 活动、本机存储和自检结果。
- 增加设置与系统自检页面，支持工作区、Codex 路径、状态栏刷新和各模块健康状态检查。
- 重做总览信息架构，并统一白天与暗色模式的背景、卡片、列表和状态色。

- Hooks 详情改为直接说明触发时机、匹配范围、实际动作、流程影响和最近观察记录；右上角增加 Hook 说明与安全启停入口。
- Hooks 列表按触发事件分组，同一事件下的每个处理器分别显示名称、匹配范围、来源和独立状态，避免多个 Hook 同名混淆。
- Skills 详情右上角增加“显式 / 隐式 / 隐藏”说明与三态切换；写入后重新读取验证，失败时恢复原启停状态和调用策略。
- Skills 列表与详情接入 macOS 系统翻译，自动将英文说明显示为中文，保留英文原文切换并缓存翻译结果。
- 重做 Codex Memory 页面：把生成文件解析为用户画像、偏好、项目、经验、注意事项和用户补充，并区分当前生效与长期记忆。
- 每条 Memory 都可搜索、复制并追溯到来源文件；来源文件独立展示，raw 与 Chronicle 不再被误当成正在生效的记忆。
- Memory 保持只读，修改时由用户复制内容后直接告诉 Codex；英文及英文为主的中英混合内容默认调用 macOS 翻译并缓存结果。
- Memory 增加全局、项目相关与待确认范围三类适用范围；解析 `applies_to` 和项目索引，直接展示关联项目名称、工作目录及判断依据。
- 将范围不明确的 Memory 标为“待确认范围”，可生成“设为全局、关联项目、不再需要”三类 Codex 整理指令；应用仍保持只读，不直接改写 Memory。
- 增加 Codex 配置备份页面，先生成脱敏预览，再由用户确认后通过本机 `gh` 上传到 GitHub 私有仓库。
- 备份默认排除 sessions、logs、cache、sqlite、auth、plugins、skills、packages 和未知运行数据，只加入用户配置快照及明确白名单文件。
- 备份上传不提供 PAT 输入框，不保存 GitHub token；依赖本机 GitHub CLI 登录状态，并拒绝公开仓库。
- 备份支持登录 GitHub、选择或创建私人仓库，并从 GitHub 提交记录恢复备份历史。
- 增加 Memory 与备份安全测试，覆盖符号链接跳过、白名单、脱敏、私有仓库检查和固定 GitHub API 参数。
- 存储清理改为分级权限：缓存可安全重建，过期临时文件和日志移入废纸篓，归档会话必须单独确认，其余数据保持只读。

## 1.1.0 - 2026-07-13

- 修正 Skills “问题”分类：只有已启用且确实存在错误或依赖缺失的 Skill 才进入该分类，隐藏状态不再被错误覆盖。
- 增加 Codex 官方限额、Token 汇总和每日趋势界面，不推测接口没有提供的费用与模型数据。
- 增加 MCP 配置、连接阶段、能力数量和安全启停界面。
- 增加 Codex Home 存储分类与缓存清理；会话、配置、凭据、Skills、插件、数据库及未知数据保持只读保护。
- 增加工作区与刷新代际校验、连接合流、配置写入串行和 App Server 进程代际保护，避免并发启动、切换或重连时混入旧状态。
- 收紧系统/托管 Skill、Hook 与 MCP 的只读边界，统一敏感文本脱敏并限制不可信协议输入。
- 增加缓存符号链接、普通文件、扫描变化与失败回滚保护。
- 区分 Skills、Hooks 与存储的“读取失败”和“确实为空”，并为 Skill/Hook 启停增加影响说明与二次确认。
- 为 App Server 事件增加连接代际，防止断开前排队的 Hook 事件混入新连接，并补充异常退出后重连测试。
- 增加隐私清单、通用架构 Release archive、ZIP/DMG、SHA-256、可选 Developer ID 签名与公证流程。
- 强化发布门禁：逐包比对 ZIP/DMG 内应用，公开模式强制 Developer ID、公证票据与 Gatekeeper 验证，失败产物不会进入 `dist/`。
- CI 固定 Xcode 16.2，同时执行测试与未签名 Release 交付物烟雾验证。
- 扩展 UI 自动化烟雾测试，逐一验证八个主要页面可从侧栏到达且不会产生重复窗口。

## 1.0.0 - 2026-07-12

- 建立原生 macOS SwiftUI 应用与中文信息架构。
- 接入 Codex App Server，支持 Skills、Hooks、通知和版本兼容。
- 支持 Skill 与 Hook 安全启停、写后验证和变更记录。
- 增加隔离的真实 Codex 集成测试、诊断、脱敏与原创应用图标。
