# Workbench 1.2.2

> 此版本已由 1.2.3 替代。1.2.2 的本地安装包不涉及运行时故障，但这一版本的源码没有通过项目固定使用的 Xcode 16.2 云端构建门禁。新用户请直接下载 1.2.3。

这个版本调整了状态栏面板的空间使用。

## 本次更新

- 状态栏面板会根据当前实际内容自动调整高度，不再固定保留一大块空白。
- 未显示 Token 活动或本机存储时，面板会随之收缩。
- 内容较多时，面板保持合理的最大高度，并通过滚动继续查看全部内容。
- 缩小分区、卡片和操作按钮之间的纵向间距，整体更接近原生 macOS 工具面板。

## 为什么被 1.2.3 替代

状态栏原来在界面和系统弹窗两处都固定为 560pt 高。Token、存储或用量信息较少时，内容虽然变少，外框仍然保持原高度，所以底部会留下大块空白。

1.2.2 第一次把它改成动态高度。本地开发环境使用的 Xcode 较新，可以正常构建；GitHub Actions 则固定使用 Xcode 16.2。该版本的 SwiftUI 会把高度监听回调按 `@Sendable` 检查，而最初实现直接在回调中读取并修改只允许主线程访问的界面状态，因此云端编译被拦截。

这不是 Workbench、ChatGPT 或 Codex 在运行时连接失败，也不代表已生成的本地安装包无法启动；问题是这一标签下的源码没有满足项目的可复现构建要求。1.2.3 已把状态更新明确切回 `MainActor`，并在同一套 Xcode 16.2 CI 中通过验证。

## 系统要求

- macOS 14 或更高版本。
- 本机已安装并登录 ChatGPT 桌面版，或已单独安装 Codex CLI。

## 下载

- `Workbench-1.2.2-15.dmg`：拖入“应用程序”安装。
- `Workbench-1.2.2-15.zip`：直接解压使用。
- `Workbench-1.2.2-15-SHA256SUMS.txt`：下载校验值。

当前社区构建没有 Apple Developer ID 签名，也没有经过 Apple 公证。首次打开时，请在 Finder 中右键 Workbench 并选择“打开”。
