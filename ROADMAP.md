# Workbench 开发路线

Workbench 的目标不是把 Codex 再包装一层，而是把原本分散、难懂、缺少状态反馈的本地能力整理成一个可信的 macOS 工具箱。

路线会随着 Codex 和 ChatGPT 桌面版变化，但有三条不会变：优先展示真实状态；不伪造 Codex 没有提供的数据；涉及配置、数据和执行权限时，让用户知道应用将要做什么。

## 正在做：自动更新

下一个版本会加入原生自动更新：

- 启动后定期从 GitHub 检查新版本。
- 在“设置 → 关于”中手动检查更新，并允许关闭自动检查。
- 展示版本说明、下载进度和安装结果。
- 更新包使用独立的 EdDSA 签名验证；即使社区版本没有付费的 Apple Developer ID，也不会安装无法验证来源的更新包。
- 保留 GitHub Release 手动下载入口。自动更新失败时，用户仍能正常下载并替换应用。

自动更新只能从首次内置更新器的版本开始生效。1.2.3 及更早版本需要最后手动更新一次。

## 接下来

- **连接诊断继续细化**：区分 ChatGPT 内置 Codex、旧版 Codex.app、独立 CLI、权限和 App Server 协议问题。
- **状态栏信息可定制**：允许调整模块顺序和显示密度，同时保持紧凑默认布局。
- **MCP 可用性验证**：在不触发高风险副作用的前提下，为只读工具提供更明确的真实调用测试。
- **Memory 阅读体验**：继续改善项目范围、来源依据、重复内容和过期信息的解释。
- **备份恢复**：在现有 GitHub 私有仓库备份记录上，增加差异预览和选择性恢复。

## 后续考虑

- 为 Skills、Hooks 和 MCP 增加更安全的导入、导出与分享方式。
- 为常见故障提供可复制的问题报告，但默认继续脱敏本地路径和配置。
- 当项目结构稳定后补充英文界面；中文体验仍然是首要目标。

## 暂不计划

- 不接入广告、遥测或独立账户系统。
- 不上传 Codex 登录凭据、会话、Memory 或完整配置到 Workbench 自建服务。
- 不为了显示一个完整数字而估算 Codex 没有返回的费用、限额或运行状态。
- 不把 Workbench 做成另一个聊天客户端；它仍然是 ChatGPT / Codex 的本地辅助工具。
